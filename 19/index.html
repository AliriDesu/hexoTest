19-JS-BOM,DOM对象

发表于 2021-08-16

DOM

描述

提供了一套用来管理控制，操作html页面的规则
是将页面中的原色，文本等都看做是节点，并由节点构成树状结构（DOM Tree）
通过管理DOM Tree来间接达到操作管理页面的目的。

DOM API

1. node.nodeName    节点名称
2. node.nodeType    节点类型

DOM Node API

规定

DOM中为了提高性能，规定如果可以获取到DOM Tree中的某一个节点，那么就可以通过这个节点直接或简介的访问到DOM Tree中的任何一个节点，可以有效的提高浏览器性能

说明

HTML页面加载的过程中存在两个严重问题，【回流reflow】和【重绘repaint】

回流 reflow

页面中元素的布局、尺寸等样式发生变化导致的页面重新加载

重绘 repaint

页面中元素的颜色或者文字颜色等样式发生变化导致的页面重新渲染。
  1. 返回当前节点的父节点 node.parentNode
  2. 返回当前节点的子节点 node.childNodes
  3. 返回当前节点的前一个兄弟节点 node.previousSibling
  4. 返回当前节点的后一个兄弟节点 node.nextSibling

    扩展

    1. 尽可能将选中操作或添加操作写在一起
    2. 合理使用文档碎片节点 documentFragment

      querySelect与getElement区别

      前者返回的结果叫：静态节点列表
      后者返回的结果叫：动态节点集合

BOM 浏览器对象模型

描述

BOM就是一套帮助js管理操作浏览器的规则，其中BOM将浏览器本身看做是一个对象，而浏览器中的某些功能模块当做是对象的属性来看待。
其中window对象为BOM规则中的顶级对象。

说明

js中所有无人认领的变量和函数都会自动成为window的属性和方法，所有window对象上的属性和方法在使用的时候，均可以省略window不写。

API

window.screen 获取浏览器加载时所在的【屏幕】的信息
window.location 获取url地址信息，其中href属性用于页面跳转
window.navigator 获取应用信息   userAgent 浏览器加载时，设备内核信息
window.history  访问过的页面的缓存

延迟器

数据本地化策略（缓存）

描述

浏览器存放数据的地方，和页面之间保持互相锁定

目的

使得浏览器中页面之间的值传递成为了可能

storage

h5新增

localStorage 本地存储仓库

描述

结构简单，只有key和value，只要不删除，就永远存在

sessionStorage 安全存储仓库

描述

结构简单与localStorage各方面都完全一致，但关闭页面数据就自动删除

补充

storage存储类型只能是字符串，如果想要想缓存中存入非基本数据类型，需要JSON转码与解析

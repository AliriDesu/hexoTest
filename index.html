12-JS-2

发表于 2021-08-04

js基本概述

描述

1. JavaScript语句的末尾使用分号表示，但不意味着每一句末尾都有分号
2. 建议一行只写一条js语句
3. js是一门面向过程的脚本式编程语言

变量

描述

创建变量的过程，实际上是向系统申请内存的过程。

扩展

内存是分区的，栈区+堆区+静态存储区，所有的变量被创建的时候，分配的内存都在栈区。

语法

var 变量名 = xxx;

命名

由字母、数字、下划线、和美元符号任意组合而成，并且不能以数字开头。

数据类型

ES5
五种基本数据类型
    数字类型        Number
    字符串类型      String
    逻辑值类型      Boolean
    未定义类型      Undefined
    空值类型        Null
    复杂数据类型    Object
ES6
新增一种基本数据类型
    标记类型        Symbol
新增五种复杂数据类型
    超级复杂复数句类型  Map
    集合类型        Set
    弱引用 超级 复杂数据类型 WeakMap
    弱引用 集合类型 WeakSet
    类 数据类型     TypedArray

运算符

描述

js中运算符表示将变量进行某种内容的变化的符号

类型

算术运算符、赋值运算符、逻辑运算符、逻辑运算符、关系运算符、三目运算符...
查看类型运算符：typeof

说明

算数运算符 + - * / %
赋值运算符 = 
1. 当运算符的左右两端的运算子均为数字类型时，仍采用基本数学运算规则
2. 当运算符的左右两端的运算子不都是数字类型时
   1. 加法会先将非字符串类型转换成字符串类型，然后再将两个字符串拼接
   2. 其余会先将字符串转换成数字类型，再按照数学逻辑运算。当字符串无法转换成数字类型的时候，运算结果就会出现NaN

扩展 三目运算符：?:

语法
表达式1 ? 表达式2 : 表达式3
名词
表达式-> 具有含义的式子
作用
判断表达式1的逻辑值，真则结果为表达式2，假则结果为表达式3

注释

名词概述

说明：JavaScript 本名 LiveScript
标准：ES5 ES6 
    js无法直接操作html页面，因为js操作的是内存，如果想要操作页面，则必须借助于某种手段来间接操作页面。这个手段就是dom。
名词：
    DOM：文档对象模型。一套用于帮助js管理操作html页面的规则。
        dom中提供了非常多的api，通过api可以打到操作页面的目的。
    API:应用程序接口。提供了一些固定的途径。通过这些途径可以完成对封闭
        系统内部的某些功能，却不需要了解内部是如何实现的。
    节点：dom规则中认为，页面的元素就是节点，文本也是节点（node）。
    节点树：dom tree，由节点构成的树状结构。
    根节点：document节点。是整个节点树的根节点。表示整个页面本身。

dom规则

api：

1. 选中页面元素
    语法：
         document.querySelector('将要选中的页面元素的选择器名称');
2. 给选中的元素添加点击事件
    语法：
        内存名称.onclick = function () {
            ...
        }
3. 给选中的元素修改内容
    语法：node.innerHTML （不可编辑标签）| node.value = xxx （可编辑标签）;
4. 给选中的元素修改样式
    语法：node.style.xxx = css样式值 | node.style.cssText = css代码
    如果出现中划线，将中划线删除，将后续的首字母大写。
5. 在内存中创建新的元素，并添加至页面中
    语法：document.createElement('创建的元素名称');
    node.appendChild(元素);

语法：var 变量名;

10-CSS媒体查询

发表于 2021-07-26

媒体查询

描述：查询媒体设备，并使得页面能够根据不同的媒体设备，加载为不同样式
语法：
    @media 生效没提设备 生效条件 （生效条件范围）{
        ...
    }
注意：
1. 要根据生效田间范围，来决定媒体查询定义的顺序
2. 语法中合理的使用空格，不能省略
特点：
优势：似的页面结构能够根据不同设备动态显示样式
劣势：一个页面中会姬成多个页面的样式代码，工作量成倍增加

flex

1.    flex
用于：flex子项
子属性：flex-grow, flex-shrink, flex-basis
取值：a) 1 → 1 1 0%
            b) auto → 1 1 auto
            c) none → 0 0 auto
            d) 0 auto → 0 1 auto 初始值
2.    flex-grow
    用于：flex子项
    取值：number， 默认为0， 不分配剩余空间
作用：用数值定义子项的扩展比率，扩展多少取决于剩余空间。
3.    flex-shrink
用于：flex子项
取值：number， 默认为1。
作用：用数值定义元素收缩比率。计算所有盒子相加比率进行收缩。
4.    flex-basis
用于：flex子项
取值：length：用长度定义宽度。Percentage：用百分比定义宽度。auto：无特定宽度，取    决于其他属性值。content：基于内容自动计算宽度。
作用：用数值定义子项的扩展比率，扩展多少取决于剩余空间
5.    flex-grow
用于：flex容器
子属性：flex-direction, flex-wrap
6.    flex-direction
用于：flex容器
取值：row,row-reverse,column,column-reverse
作用：定义主轴方向
7.    flex-wrap
用于：flex容器
取值：nowrap,wrap,wrap-reverse
作用：设置容器内不是否换行
8.    align-content
用于：多行弹性盒模型容器
取值：    a）flex-start：从起始位置随侧轴堆叠
b）flex-end：在容器结束为止随侧轴堆叠
c）center：随侧轴中间堆叠
d）space-between：在容器中平均分布，两端没有空隙
e）space-around：在容器中平均分布，两端有空隙
f）stretch：各部分伸展占用剩余空间，剩余空间为负数则等同于flex-start
作用：设置多行模型内部盒子侧轴堆叠方式
9.    align-items
用于：弹性盒模型容器
取值：    a）flex-start：从起始位置随侧轴堆叠
b）flex-end：在容器结束为止随侧轴堆叠
c）center：随侧轴中间堆叠
d）stretch：各部分伸展占用剩余空间，剩余空间为负数则等同于flex-start
e）baseline：元素盒与基线对齐，弱行内轴与侧轴为同一条，则等同于flex-start
作用：设置单行模型内部盒子侧轴堆叠方式
10.    align-self
用于：flex子项
取值：    a）flex-start：从起始位置随侧轴堆叠
b）flex-end：在容器结束为止随侧轴堆叠
c）center：随侧轴中间堆叠
d）stretch：各部分伸展占用剩余空间，剩余空间为负数则等同于flex-start
e） baseline：元素盒与基线对齐，弱行内轴与侧轴为同一条，则等同于flex-start
f） auto：计算父元素的align-items值，否则为stretch
作用：设置盒子堆叠方式
11.    justify-content
用于：flex容器
取值：    a）flex-start：从起始位置随侧轴堆叠
b）flex-end：在容器结束为止随侧轴堆叠
c）center：随侧轴中间堆叠
d）space-between：在容器中平均分布，两端没有空隙
e）space-around：在容器中平均分布，两端有空隙
作用：设置盒子元素在主轴上的对齐方式
12.    flex-grow
    用于：flex子项和flex容器中的绝对定位子元素
    取值：integer
    作用：用整数来定义排列顺序，数值小的会被盖住，可以为负值

07-盒子模型

发表于 2021-07-13 更新于 2021-07-15

CSS盒子模型使用问题

1. 垂直 相邻 块元素 margin会发生合并，取最大值（不是bug是特性）
   1. padding可以计算背景颜色
2. 垂直 相邻 父子级 元素高度合并（高度击穿，是bug）
   1. 原因：子级元素无法确定父级元素的边界
   2. 解决办法：overflow：auto 伪元素：display：table
3. 水平方向元素边距问题
    水平不合并
4. 边距负值问题
   1. margin减小但大于0时，会想着元素边界移动
   2. margin越过0，继续减小，元素会想着正值的反向移动
5. 高度塌陷
   1. 父级float，父级脱离文档流
   2. overflow：hidden，对类似下拉菜单不友好
   3. display：inline-block，变成类文本效果
   4. position：absolute，需要自己定位

文档流

浮动：流水排列，半脱离文档流
绝对定位：完全脱离文档流

2. 在父级伪元素添加clear：both

04-css基础2

发表于 2021-07-12

文档流

页面中元素从上到下，从左到右构成的文档结构。

浮动

描述：元素脱离文档流显示
语法：float：left/right
本质：为了实现文字环绕效果，只能遮挡结构，文本会四周环绕
说明：
    1. float：相对于浮动元素所在的容器靠左/右
    2. 浮动属性不能修改元素的位置
    3. 浮动元素不是行元素，也不是块元素，称为浮动元素，浮动元素大小由本身决定
注意：
    1.浮动元素是处于同一层级的，所有的浮动元素均排成一排，换行是因为容器不够
    2.页面开发原则：尽可能保证在一个容器内部的修改，不要影响到容器外部
    3.清除浮动
        描述：将浮动元素对页面造成的影响清除，使得浮动元素在文档流中仍然可以占有大小结构，但浮动元素造成的元素变化不会还原。
        语法：在浮动元素的后面添加空白的会计元素，并给这个空白块级元素添加css样式：clear:both。

定位

描述：选定一个参照物，并使得元素根据参照物进行位置移动
语法：position：
            static：    静态定位（默认，无定位）
            relative：  相对定位（设置当前元素为定位参照物）
            absolute：  绝对定位（设置当前元素脱离文档流，并准备相对参照物开始定位）    TRBL
说明：
    1. 如果没有设定relative，absolute元素默认根据浏览器定位
    2. 如果当前absolute的祖先级元素有多个relative，则就近原则
    3. 如果当前absolute元素不是最外层元素，那么只要它的父级中position不为static，就根据父级定位
    4. 如果当前absolute祖先级元素中不存在relative，那么默认根据浏览器定位
注意：
    定位元素采用堆叠方式

z-index：用来控制定位元素的堆叠顺序
        规则：
            同父原则：如果元素是同一个父级的子级，谁的z-index大，谁在上。
            从父原则：谁的父元素z-index大，谁的子级就在上
fixed：窗口定位（固定定位，锁定定位）
sticky：粘性定位，由前四个组合
